文件操作

文件描述符

在 POSIX 系统上，对于每个进程，内核都维护着一张当前打开着的文件和资源的表格。 每个打开的文件都分配了一个称为文件描述符的简单的数字标识符。 在系统层，所有文件系统操作都使用这些文件描述符来标识和跟踪每个特定的文件。 Windows 系统使用了一个虽然不同但概念上类似的机制来跟踪资源。 为了简化用户的工作，Node.js 抽象出操作系统之间的特定差异，并为所有打开的文件分配一个数字型的文件描述符。

异步打开文件

fs.open()方法用于分配新的文件描述符。一旦被分配，则文件描述符可用于从文件读取数据、向文件写入数据、或请求关于文件的信息。

当文件操作完成时需要关闭描述符，否则将导致内存泄漏，最终导致应用程序崩溃。

const fs = require('fs');

fs.open('./file/test.txt', 'w', (err, fd) => {
    console.log(fd);//fd 是文件描述符
   if(err) throw err;
   //使用文件属性
   
   //关闭文件操作符
   fs.close(fd,(err)=>{
      if(err) throw err;
   })
})
同步打开文件

fs.openSync()方法打开文件效率不如异步的高，需要等待文件打开完才能执行后续的任务，该方法会返回文件描述符(而不是在回调中提供)：

const fs = require('fs');

try {
  const fd = fs.openSync('/file/test.txt', 'r')
} catch (err) {
  console.error(err)
}

console.log("我被执行了");// 如果用fs.open()方法，这一步的打印结果会在文件打开之上

上面的w作为fs.open()的第二个参数，类似的标志有如下几种：

  • r+：打开文件用于读写
  • w+：打开文件用于读写，将流定位到文件的开头。如果文件不存在则创建文件
  • a：打开文件用于写入，将流定位到文件的末尾。如果文件不存在则创建文件
  • a+：打开文件用于读写，将流定位到文件的末尾。如果文件不存在则创建文件

文件属性

fs.stat()方法调用时传入文件路径，获得文件的详细信息时会调用传入的回调函数，并带上两个参数：错误消息和文件属性。

//异步
const fs = require('fs')

fs.stat('/Users/joe/test.txt', (err, stats) => {
  if (err) {
    console.error(err)
    return
  }
  //可以访问 `stats` 中的文件属性
})

//同步
const fs = require('fs')

try {
  const stats = fs.statSync('/Users/joe/test.txt')
} catch (err) {
  console.error(err)
}

文件属性一些常用的方法：

const fs = require('fs')
fs.stat('/file/test.txt', (err, stats) => {
  if (err) {
    console.error(err)
    return
  }

  stats.isFile() //true，判断文件是否是文件
  stats.isDirectory() //false，判断文件是否是目录
  stats.isSymbolicLink() //false，判断文件是否符号链接
  stats.size //1024000 //= 1MB，获取文件的大小，以字节为单位
})

文件路径

Windows与Linux和macOS系统里面的文件路径存在差异，因此需要使用一下方式将此模块引入到文件中,然后使用其他方法：

const path = require('path')
从路径中获取信息
  • dirname：获取文件的父文件夹
  • basename：获取文件名部分
  • extname：获取文件的扩展名部分
const path = require('path')

const notes = '/file/test.txt'
 
path.dirname(notes)// /file
path.basename(notes)// test.txt
path.extname(notes)// .txt

//通过为basename指定第二个参数来获取不带扩展名的文件名
path.basename(notes, path.extname(notes))// notes
使用路径

  • path.join()方法可以连接路径的两个或多个片段

    const name = 'joe'
path.join('/', 'users', name, 'notes.txt') // \users\ws\notes.txt

  • path.resolve()方法可以获得相对路径的绝对路径计算

    path.resolve('test.txt') // \node\test\test.txt

//上面的方法知识简单的将/test.txt附加到当前工作目录
//如果指定第二个文件夹参数，则该方法会使用第一个作为第二个的基础：
path.resolve('tmp', 'test.txt')// \node\test\tmp\test.txt

//若第一个参数以斜杠开头，则表示它是绝对路径：
path.resolve('/etc', 'test.txt')// \etc\test.txt

  • path.normalize()当包含. ..或双斜杠之类的相对说明符时，会尝试计算实际的路径：

    path.normalize('/users/ws/..//test.txt') // \users\test.txt

读取文件与写入文件

读取文件

fs.readFile()方法向其传入路径 、编码、以及会带上文件数据（以及错误）进行调用的回调函数

//异步
const fs = require('fs')

fs.readFile('./file/test.txt', 'utf8', (err, data) => {
    if (err) {
        console.error(err)
        return
    }
    console.log(data) //此处打印文件里面的内容
})

//同步
const fs = require('fs')

try {
  const data = fs.readFileSync('./file/test.txt', 'utf8')
  console.log(data) //此处打印文件里面的内容
} catch (err) {
  console.error(err)
}
写入文件

fs.readFile()方法可以往指定路径里面写入内容，默认情况下改方法会替换文件的内容，可以通过指定标志来修改默认的行为，更多标志请查看：更多标志

//异步
const fs = require('fs')

const content = '我正在写入内容'

fs.writeFile('./file/test.txt', content, { flag: 'a+' }, err => {
    if (err) {
        console.error(err)
        return
    }
    console.log('文件写入成功');
})

//同步
const fs = require('fs')

const content = '我正在写入内容'

try {
  const data = fs.writeFileSync('./file/test.txt', content)
  console.log('文件写入成功');
} catch (err) {
  console.error(err)
}

fs.appendFile()方法能更便捷的将内容追加到文件末尾(对应的同步操作为fs.appendFileSync())

文件夹相关操作

  • fs.accrss()：检查文件夹是否存在以及NodeJS是否具有访问权限

  • fs.mkdir()或fs.mkdirSync()：创建新的文件夹

  • fs.readdir()或fs.readdirSync()：读取文件夹里目录内容

    const fs = require('fs')
const path = require('path')

//读取文件夹的内容（全部的文件和子文件夹），并返回它们的相对路径：
const folderPath = '/前端技术栈汇总/node/test'
console.log(fs.readdirSync(folderPath))

//获取完整的路径：
let a = fs.readdirSync(folderPath).map(fileName => {
    return path.join(folderPath, fileName)
})
console.log(a)

//也可以过滤结果以仅返回文件（排除文件夹）：
const isFile = fileName => {
    return fs.lstatSync(fileName).isFile()
}
let b = fs.readdirSync(folderPath).map(fileName => {
    return path.join(folderPath, fileName)
}).filter(isFile)
console.log(b)

  • fs.rename()或fs.renameSync()：重命名文件夹，第一个参数是当前的路径，第二个参数是新的路径

  • fs.rmdir()或fs.rmdirSync()：可以删除空文件夹。删除包含内容的文件夹会复杂点，一般安装fs-extra模块，使用里面的remove()方法能更好的删除文件夹

    const fs = require('fs-extra')

const folder = '/Users/joe'

fs.remove(folder, err => {
  console.error(err)
})

//与promise一起使用
fs.remove(folder)
  .then(() => {
    //完成
  })
  .catch(err => {
    console.error(err)
  })

//使用async/await
async function removeFolder(folder) {
  try {
    await fs.remove(folder)
    //完成
  } catch (err) {
    console.error(err)
  }
}

const folder = '/Users/joe'
removeFolder(folder)

文件系统模块

  • fs.access(): 检查文件是否存在，以及 Node.js 是否有权限访问。
  • fs.appendFile(): 追加数据到文件。如果文件不存在，则创建文件。
  • fs.chmod(): 更改文件（通过传入的文件名指定）的权限。相关方法：fs.lchmod()、fs.fchmod()。
  • fs.chown(): 更改文件（通过传入的文件名指定）的所有者和群组。相关方法：fs.fchown()、fs.lchown()。
  • fs.close(): 关闭文件描述符。
  • fs.copyFile(): 拷贝文件。
  • fs.createReadStream(): 创建可读的文件流。
  • fs.createWriteStream(): 创建可写的文件流。
  • fs.link(): 新建指向文件的硬链接。
  • fs.mkdir(): 新建文件夹。
  • fs.mkdtemp(): 创建临时目录。
  • fs.open(): 设置文件模式。
  • fs.readdir(): 读取目录的内容。
  • fs.readFile(): 读取文件的内容。相关方法：fs.read()。
  • fs.readlink(): 读取符号链接的值。
  • fs.realpath(): 将相对的文件路径指针（.、..）解析为完整的路径。
  • fs.rename(): 重命名文件或文件夹。
  • fs.rmdir(): 删除文件夹。
  • fs.stat(): 返回文件（通过传入的文件名指定）的状态。相关方法：fs.fstat()、fs.lstat()。
  • fs.symlink(): 新建文件的符号链接。
  • fs.truncate(): 将传递的文件名标识的文件截断为指定的长度。相关方法：fs.ftruncate()。
  • fs.unlink(): 删除文件或符号链接。
  • fs.unwatchFile(): 停止监视文件上的更改。
  • fs.utimes(): 更改文件（通过传入的文件名指定）的时间戳。相关方法：fs.futimes()。
  • fs.watchFile(): 开始监视文件上的更改。相关方法：fs.watch()。
  • fs.writeFile(): 将数据写入文件。相关方法：fs.write()。

路径模块

  • path.basename()：返回路径的最后一部分，第二个参数可以过滤掉文件的扩展名

  • path.dirname()：返回路径的目录部分

    require('path').dirname('/test/file/test.txt') // '/test/file'

  • path.extnama()：返回路径的扩展名部分，若无则返回空

  • path.isAbsolute()：如果是绝对路径返回true

  • path.normalize()：当包含.、..或双斜杠相对的说明符是，则尝试计算实际的路径

  • path.parse()：解析对象的路径为组成其的片段

    require('path').parse('/users/test.txt')

//结果如下
{
  root: '/', //根路径
  dir: '/users', //从根路径开始的文件夹路径
  base: 'test.txt', //文件名+扩展名
  ext: '.txt', //文件名
  name: 'test' //文件扩展名
}

  • path.relative()：接收两个路径作为参数。基于当前工作目录，返回第一个路径到第二个路径的相对路径

  • path.resolve()：获得相对路径的绝对路径计算

事件

引入事件模块
const Events = require('events')

//声明一个事件触发器
const door = new Events()
创建事件监听
  • emitter.on()：同emitter.addListener()，添加当事件被触发时调用的回调函数
  • emitter.once()：同上，但该回调函数只会调用一次，后不再调用
  • emitter.prependListener()：当使用 on 或 addListener 添加监听器时，监听器会被添加到监听器队列中的最后一个，并且最后一个被调用。 使用 prependListener 则可以在其他监听器之前添加并调用。
  • emitter.prependOnceListener()：同上
触发监听事件
  • emitter.emit()：触发事件。 按照事件被注册的顺序同步地调用每个事件监听器
清除事件监听

  • emitter.removeListener()：移除特定的监听器。 可以通过将回调函数保存到变量中（当添加时），以便以后可以引用它

    const doSomething = () => {}
door.on('open', doSomething)
door.removeListener('open', doSomething)

  • emitter.removeAllListeners()：移除对象的所有监听特定事件的监听器

Buffer

Buffer是内存区域。其被引入用以帮助开发者处理二进制数据，在此生态系统中传统上只处理字符串而不是二进制数据。因此效率更快。
