三、JS基本语法

1、变量
  • 变量声明：var
  • 命名规则：
    1. 变量名必须以英文字母、下划线、$开头
    2. 变量名可以包括英文字母、下划线、$、数字
    3. 不可以用系统的关键字、保留字作为变量名
2、值类型

  • 原始值——存在栈stack里面

    Number：数字

    Boolean：布尔类型，表示逻辑，ture/false

    String：字符串

    undefined：没有定义的值

    null：空，占一个位置

  • 引用值——存在堆heap里面

    array、object、function、date、RegExp….

3、js语句基本规则
  • js语法错误会引发后续代码终止，但不会影响其他js代码块
  • 格式规范：语句后面加分号，=、+、-两边都应该有空格
4、运算操作符

  • 赋值的顺序，自右向左；计算的顺序，自左向右

  • “+”：

    • 数学运算、字符串相加就是字符串连接
    • 任何数据类型加字符串都等于字符串

  • “%”：取余数

  • “++”:a=a+1就是a++。a++先执行表达式再自增，执行表达式时使用的是a的原值；++a先自增再执行表达式，执行表达式时使用的是自增后的a

    var a = 1;
document.write(a ++); //1
document.warite(a); //2

var a = 1;
var b = a ++ + 1;
document.write(a); //2
document.write(b); //2

  • “+=”:a+=10即是a=a+10

  • “,”：逗号操作运算符，先看逗号前面的表达式并计算，然后看逗号后面的表达式并计算，然后返回后面表达式的结果。

    var a = (2, 3);
console.log(a);     //3
5、比较运算符

  • ```javascript
    var a = ‘10’ > ‘8’;//只比较第一个字符
    document.write(a); //false

    
- ```javascript
  var a = Infinity == Infinity;
  document.write(a);  //true
  
  var a = NaN == NaN;
  document.write(a);  //false   特例

  • &&（与）：最后会返回一个运算结果。先看第一个表达式转换为第一个布尔值的结果，如果布尔值为真，那么它会看第二个表达式转换为布尔值的结果，如此往后直到布尔值为假时返回该值；如果只有两个表达式的话，只看第二个表达式，就可以返回该表达式的值。

    var a = 0 && 2;//undefined,null,NaN,"",0,false 转换为布尔值为faslse
document.write(a); //0

  • a || b（或）：碰到真就返回真的值。

  • ！（非）：布尔值取反

##### 8、显式类型转换

- Number():返回number类型的值。undefined返回NaN,null返回0。

- parseInt():返回整数，小数直接去掉；所有非数字类型都将返回NaN。parseInt(num,n)中n代表num的进制，然后将该数转换为10进制，n的取值为2-36。

  ```javascript
  var a = 101010;
  var b = parseInt(a,2);
  console.log(b);//42 
  
  var demo = "123abc";
  var num = parseInt(demo);
  console.log(typeof(num) + ":" + num); //number : 123    从左到右以数字为开始一直到非数字位结束，把数字位返回

  • String():转换为字符串

  • toString():用法为要转换的数据.toString()，unfefind、null不能使用该操作。也可用作进制转换：

    var a = 10;
console.log(a.toString(8)); //12
    9、隐式类型转换

  • isNaN():会将括号里面的先放到Number()返回值，再判断它是否是NaN，返回true或者false

  • ++/– –、+/-(一元正负，如+a，-a) ：a++，先将a放入Number()执行再加减。

  • 加：会调用toString();

  • 减乘除：调用Number();

  • && || !:调用Boolean()进行类型转换；

  • < > <= >=：如字符串和数字比较会把字符串转换成数字类型；

  • == !=：如1 == “1” == true，返回的值为布尔类型。

3、作用域

  • **[[scope]]:**每个JavaScript函数都是一个对象，对象中有些属性我们可以访问，但有些不可以，这些属性进攻JavaScript引擎存取，[[scope]]就是其中一个，[[scope]]指的就是作用域，其中存储了运行期上下文的集合。

  • 作用域链：[[scope]]中锁储存的执行期上下文对象的集合，这个集合呈链式链接，我们把这种链式链接叫做作用域链。

    function a() {
	function b() {
      var b = 234
   }
   var a = 123;
   b();
}
var glob = 100;
a();
//当函数a被定义时，在作用域链中a.[[scope]]里第0为为GO{}
//当函数a被执行时，a.[[scope]]里第零位变为AO{},第一位变为GO{}，如下图：scope chain即是作用域链
//当函数b被定义时，b的[[scope]]里面存的是a的作用域链(图3)，b产生的AO{}和a产生的AO{}是相同的

  • 运行期上下文：当函数正要执行前一刻时，会创建一个称为执行期上下文的内部对象。一个执行期上下文定义了一个函数执行时的环境，函数每次执行时对应的执行上下文都是独一无二的，所以多次调用一个函数会导致创建多个执行上下文，当函数执行完毕，它锁产生的执行上下文被销毁。

  • 查找变量：从作用域链的顶端依次向下查找。

六、预编译

1、JS执行三部曲

  • 语法分析：将所有代码扫描一遍看是否存在语法错误

  • 预编译：发生在函数执行的前一刻。

    • 函数体系里的预编译：
    1. 创建AO对象（activation object），执行期上下文；
    2. 找形参和变量声明，将变量和形参名作为AO属性名，值为undefined；
    3. 将实参和形参值相统一；
    4. 在函数里面找函数声明，值赋予函数体（覆盖前面的值）；

    • 全局里的预编译：

      • 对应上面的少了第三步，第一步变为生成GO对象（global object）。

    • imply global 暗示全局变量：即任何变量，如果变量未经声明就赋值，此变量就为全局对象（window）所有。

    a = 10;——> window.a = 10;

function test() {
   var a = b = 123;//b为暗示全局变量
}
test();
console.log(window.a);//undefined,a是局部变量
console.log(window.b);//123，b是未经声明就赋值的变量
    • 一切声明的全局变量，全是window的属性。window就是全局的域。
    var a = 123; ——> window.a = 123;

七、闭包

  • 当内部函数被保存到外部时，将会生成闭包。如下面的代码会产生闭包，return b将b的引用存到了全局里面，执行后函数a的AO会被销毁，但b的作用域链存着a的AO和GO。

    ex1:
function a() {

	function b() {
		var bbb = 234;
		document.write(aaa);
	}
	
	var aaa = 123;
	return b;
}
var demo = a();
demo(); // 123

ex2:
function a() {
   
   var num = 100;
  	function b() {
      num ++;
      console.log(num);
   }
   return b;
}
var demo = a();
demo();               //101
demo();               //102

ex3:
function test() {
   var num = 100;
   function a() {
      num ++ ;
      console.log(num);
   }
   function b() {
		num -- ;
      console.log(num);
   }
   return [a,b]
}
var myArr = test();
myArr[0]();				//101
myArr[1]();				//100,函数a和函数b用的时同一个AO
  • 闭包的危害：当内部函数被保存到外部时，将会生成闭包。闭包会导致原有作用域链不释放，造成内存泄漏。（占内存越多，可用内存就越少）

  • 闭包的作用：

    • 实现共有变量

      • eg：函数累加器，ex1

    • 可以做缓存（储存结构）

      • eg：ex3

    • 可以实现封装，属性私有化。

      • eg：

        //如下形成的闭包，Mid变成了该函数的私有化变量。
var inherit = (function () {
	var Mid = function () {};//在函数执行后销毁，为什么return到外面的函数还能用函数里面的变量？
	return function (Target, Origin) {
      //应为该函数被保存到了外部，形成了闭包，保存了原函数的执行期上下文，里面就包括Mid
      Mid.prototype = new Mid();
      Target.prototype = new Mid();
      Target.prototype.constuctor = Target;
      Target.prototype.uber = Origin.prototype
	}
}());

    • 模块化开发，防止污染全局变量。
